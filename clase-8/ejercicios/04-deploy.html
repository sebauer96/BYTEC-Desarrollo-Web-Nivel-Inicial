DESAFÍO FINAL

🚀 Publicá tu Sitio Web

Guía paso a paso para hacer deploy en GitHub Pages

01

Crear cuenta en GitHub

Andá a github.com y creá una cuenta gratis.

  • Click en "Sign up"
  • Ingresá tu email, una contraseña y un nombre de usuario
  • Verificá tu email (te llega un código)
  • Elegí el plan Free
💡 Tu nombre de usuario va a aparecer en la URL pública. Elegí algo profesional, sin números raros ni símbolos.
02

Crear un nuevo repositorio

El "repositorio" o "repo" es donde va a vivir tu código en GitHub.

  • Click en el ícono + arriba a la derecha → "New repository"
  • Ponele un nombre descriptivo (ej: mi-portafolio, pixel-cafe)
  • Dejá la descripción opcional
  • Visibility: Public (necesario para Pages gratis)
  • NO marques "Add README", "Add .gitignore" ni nada más
  • Click "Create repository"
⚠️ El nombre del repo va a aparecer en la URL final. mi-sitio es mejor que proyecto-final-clase-8-final-FINAL.
03

Subir tu código (opción fácil: web)

Si nunca usaste Git, podés subir los archivos desde la web:

  • En tu repo recién creado, click "uploading an existing file"
  • Arrastrá TODOS los archivos de tu proyecto (HTML, CSS, imágenes)
  • Asegurate que el archivo principal se llame index.html
  • Escribí un mensaje de commit (ej: "Primer subida")
  • Click "Commit changes"
📂 Si tu proyecto usa subcarpetas (ej: img/), se mantienen al subir. Solo arrastrá la carpeta entera.
03b

Subir tu código (opción avanzada: Git)

Si querés aprender Git desde el principio, este es el camino profesional:

# Abrí terminal/CMD en la carpeta de tu proyecto

git init
git add .
git commit -m "Primer commit"

# GitHub te muestra exactamente estos comandos en el repo vacío:
git remote add origin https://github.com/TUUSUARIO/NOMBRE-REPO.git
git branch -M main
git push -u origin main

# La próxima vez que cambies algo:
git add .
git commit -m "Descripción del cambio"
git push
🛠️ VS Code tiene Git integrado. Mirá el ícono Source Control (⎇) en la barra lateral.
04

Activar GitHub Pages

Ahora que tu código está en GitHub, vamos a publicarlo:

  • En tu repo, click "Settings" (arriba a la derecha)
  • En el menú lateral izquierdo, scrollea hasta "Pages"
  • En "Source" elegí "Deploy from a branch"
  • En "Branch" elegí main y carpeta / (root)
  • Click "Save"
⏰ GitHub tarda 1-2 minutos en publicar. Después aparece una banda verde arriba con tu URL.
05

¡Visitar tu sitio!

Tu URL va a tener este formato:

https://TUUSUARIO.github.io/NOMBRE-REPO/

Algunos ejemplos:

  • https://martinacode.github.io/mi-portafolio/
  • https://juangamer.github.io/pixel-cafe/
  • https://luciapix.github.io/banda-website/
🎉 Compartí tu URL en redes sociales, agregala a tu CV, mostrasela a tus amigos. ¡Acabás de publicar tu primera web en internet!
06

Actualizar tu sitio

Cada vez que cambies algo, repetí estos pasos:

  • Web: ir al repo → click en el archivo → ícono lápiz → editar → commit
  • Git: git add . → git commit -m "..." → git push

GitHub Pages vuelve a publicar automáticamente en 1-2 minutos.

🎁 Bonus: Tips Profesionales

📛 Custom Domain

Si comprás un dominio (ej: tunombre.com), podés conectarlo a GitHub Pages gratis. En Settings → Pages → Custom domain.

🔒 HTTPS

GitHub Pages te da SSL gratis. Activá la opción "Enforce HTTPS" en Settings → Pages.

📊 Analytics

Agregá Google Analytics o Plausible para ver cuánta gente visita tu sitio.

🌍 Otras opciones

Si querés explorar más, probá Netlify (drag & drop) o Vercel (mejor para apps modernas). Ambos gratis.
