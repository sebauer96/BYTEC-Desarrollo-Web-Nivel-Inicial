✨ ¿Por Qué Animar?

Las animaciones bien hechas elevan la experiencia de tu sitio. No son decoración: son comunicación visual:

Beneficios

  • Guían la atención del usuario
  • Dan feedback al hacer click/hover
  • Hacen el sitio se sienta "vivo"
  • Comunican estados (loading, error)
  • Mejoran la percepción de calidad

⚠️ Cuidado con

  • Animaciones largas (>500ms) cansan
  • Demasiadas animaciones distraen
  • Animar properties caras (top, left)
  • Olvidar usuarios sensibles al movimiento

🧠 Regla de oro: las mejores animaciones son las que casi no notás. Sutil > exagerado.

🎚️ transition: La Animación Más Simple

transition hace que el cambio entre dos estados sea suave en vez de instantáneo. Perfecto para hover effects:

/* Sin transition: cambio instantáneo */
.boton {
  background: #f75001;
}
.boton:hover {
  background: #04589d;
}

/* Con transition: cambio suave de 0.3s */
.boton {
  background: #f75001;
  transition: background 0.3s;     /* propiedad duración */
}
.boton:hover {
  background: #04589d;
}

/* Animar varias propiedades a la vez */
.card {
  transition: transform 0.3s, box-shadow 0.3s, border-color 0.3s;
}

/* O todas las que cambien */
.card {
  transition: all 0.3s;
}

⏱️ Duraciones recomendadas: 150ms-300ms para hover, 200ms-500ms para transformaciones más grandes. Más tiempo se siente lento.

🔄 transform: Mover, Rotar, Escalar

transform permite manipular elementos de formas que position no puede. Es performante (no causa reflow):

Función ¿Qué hace? Ejemplo
translate Mueve el elemento translate(20px, 0)
translateY Mueve solo en eje Y translateY(-4px)
scale Cambia el tamaño scale(1.05)
rotate Rota en grados rotate(45deg)
skew Inclina el elemento skew(10deg) 
/* Hover effect clásico: levantar la card */
.card {
  transition: transform 0.3s, box-shadow 0.3s;
}

.card:hover {
  transform: translateY(-6px);
  box-shadow: 0 12px 30px rgba(0,0,0,0.2);
}

🎯 Hover Effects Profesionales

Estos son los efectos hover más usados en webs profesionales:

1. Lift (Levantar)

.card {
  transition: all 0.3s;
}
.card:hover {
  transform: translateY(-4px);
  box-shadow: 0 10px 25px rgba(0,0,0,0.15);
}

2. Scale (Zoom)

img {
  transition: transform 0.3s;
}
img:hover {
  transform: scale(1.05);
}

3. Glow (Brillo)

.btn {
  transition: box-shadow 0.3s;
}
.btn:hover {
  box-shadow: 0 0 20px rgba(247, 80, 1, 0.6);
}

4. Color Shift

a {
  color: #888;
  transition: color 0.2s;
}
a:hover {
  color: #f75001;
}

🎬 @keyframes y animation

Para animaciones más complejas (que no son hover), usás @keyframes + animation. Definís los pasos y CSS interpola:

/* 1. Definir los pasos de la animación */
@keyframes fadeIn {
  from {
    opacity: 0;
    transform: translateY(20px);
  }
  to {
    opacity: 1;
    transform: translateY(0);
  }
}

/* 2. Aplicarla a un elemento */
.hero-content {
  animation: fadeIn 0.8s ease-out;
}

/* Sintaxis completa de animation */
.elemento {
  animation: nombre duración timing-function delay iteración;
}

/* Ejemplo: pulso infinito */
@keyframes pulse {
  0%, 100% { transform: scale(1); }
  50%      { transform: scale(1.1); }
}

.notification-dot {
  animation: pulse 1.5s ease-in-out infinite;
}

💡 Diferencia clave: transition = entre dos estados (hover). animation = secuencia de pasos definida (loops, entradas, etc.).

📚 Animaciones Útiles para Copiar

Estas animaciones las podés usar tal cual en cualquier proyecto:

/* Fade In desde abajo */
@keyframes fadeInUp {
  from { opacity: 0; transform: translateY(30px); }
  to   { opacity: 1; transform: translateY(0); }
}

/* Slide desde la izquierda */
@keyframes slideInLeft {
  from { opacity: 0; transform: translateX(-50px); }
  to   { opacity: 1; transform: translateX(0); }
}

/* Bounce de entrada */
@keyframes bounceIn {
  0%   { opacity: 0; transform: scale(0.3); }
  50%  { opacity: 1; transform: scale(1.05); }
  70%  { transform: scale(0.9); }
  100% { transform: scale(1); }
}

/* Spinner para loading */
@keyframes spin {
  from { transform: rotate(0deg); }
  to   { transform: rotate(360deg); }
}

.loader {
  animation: spin 1s linear infinite;
}

/* Cursor parpadeando (como el _ de BYTEC) */
@keyframes blink {
  0%, 50%   { opacity: 1; }
  51%, 100% { opacity: 0; }
}

💎 Pulido Visual: Los Detalles que Importan

Esos pequeños detalles separan una web "ok" de una profesional:

🌑

box-shadow

0 4px 20px rgba(0,0,0,0.1) agrega profundidad sutil a cards y botones.

⭕

border-radius

8-12px en cards. 50% para avatares circulares. 20px+ para botones tipo "pill".

✨

focus states

Inputs y botones deben tener un :focus visible para accesibilidad con teclado.

🖱️

cursor: pointer

En todo lo que sea clickeable (botones custom, cards interactivas).

📜

scroll-behavior

html { scroll-behavior: smooth } hace que los anchor links se animen.

🎨

::selection

Personalizá el color cuando el usuario selecciona texto en tu sitio.

🌐 ¿Qué es "Publicar" un Sitio?

Publicar (deploy) = subir tus archivos a un servidor en internet para que cualquiera pueda visitarlos con una URL:

💻

Tu compu

Los archivos viven en tu carpeta local.

→
📦

Git Push

Subís el código a GitHub.

→
🌐

GitHub Pages

GitHub te da una URL gratis: tuusuario.github.io/proyecto

🆓 GitHub Pages es 100% gratis para sitios estáticos (HTML/CSS/JS). Perfecto para tu portafolio o cualquier proyecto chico.

📦 Git: Lo Mínimo que Necesitás

Git es un sistema para versionar código. GitHub es la "nube" donde lo guardás:

# Configurá Git la primera vez (solo una vez)
git config --global user.name "Tu Nombre"
git config --global user.email "tu@email.com"

# En la carpeta de tu proyecto:
git init                        # Crear el repo local
git add .                       # Marcar todos los archivos
git commit -m "Primer commit"   # Guardar la versión

# Conectar con GitHub (después de crear el repo en github.com)
git remote add origin https://github.com/USUARIO/PROYECTO.git
git branch -M main
git push -u origin main         # ¡Subir todo!

💡 VS Code tiene Git integrado: ícono de "Source Control" (⎇) en la barra lateral. Podés hacer commit y push con clicks sin la terminal.

🐙 Crear el Repo en GitHub

Pasos para crear tu repo y subir tu proyecto:

1

Andá a github.com y creá una cuenta gratis si no tenés.

2

Click en el "+" arriba a la derecha → "New repository".

3

Ponele un nombre (ej: mi-portafolio), dejá Public y NO marques nada más.

4

GitHub te muestra los comandos para conectar tu carpeta local con el repo remoto.

El nombre importa

El nombre del repo va a aparecer en la URL final: tuusuario.github.io/nombre-repo

Public vs Private

Para usar GitHub Pages gratis, el repo tiene que ser Public. Cualquiera puede ver el código.

🚀 Activar GitHub Pages

Una vez subido el código, activar Pages toma 30 segundos:

1

En tu repo

Andá a Settings (arriba a la derecha del repo).

→
2

Pages

En el menú lateral, buscá "Pages".

→
3

Branch: main

En "Branch" elegí main y carpeta / (root). Click "Save".

⏰ Esperá 1-2 minutos. Después tu sitio está vivo en: https://TUUSUARIO.github.io/NOMBRE-REPO/

🌍 Otras Plataformas para Deploy

GitHub Pages es la más simple, pero hay otras alternativas gratis y excelentes:

🐙 GitHub Pages

  • 100% gratis
  • Conectado a tu repo
  • Solo HTML/CSS/JS
  • URL: *.github.io
Recomendado para principiantes

🪂 Netlify

  • Drag & drop (sin Git)
  • Deploy automático en cada push
  • SSL y CDN incluidos
  • URL: *.netlify.app
Profesional y fácil

▲ Vercel

  • Mejor para apps modernas
  • Optimizado para frameworks
  • Deploy en cada commit
  • URL: *.vercel.app
Para proyectos avanzados

✅ Checklist Antes de Publicar

Antes de hacer deploy, repasá estos puntos para que tu sitio se vea profesional:

📝 El archivo principal se llama index.html (sí o sí)
🔗 Todos los links internos funcionan
🖼️ Las imágenes tienen alt="" descriptivo
📱 Probaste en mobile, tablet y desktop
🎨 Los colores tienen buen contraste (WCAG AA)
🏷️ El <title> es claro y descriptivo
📋 Agregaste un <meta name="description"> en el head
🚫 No tenés console.log o código de prueba
⚡ Las imágenes están optimizadas (no super pesadas)
🌐 Probaste con Live Server en VS Code antes de subir
