🌐 ¿Qué es la Web?

La World Wide Web es un sistema de documentos interconectados que viven en servidores alrededor del mundo. Cuando abrís una página, estás pidiendo un archivo a otra computadora.

📱

Vos (Cliente)

Tu navegador pide una página web.

⇄
🖥️

Servidor

Una computadora remota envía los archivos.

→
🌐

Página Web

Tu navegador interpreta los archivos y los muestra.

💡 Cuando escribís www.google.com estás enviando un pedido a un servidor de Google que responde con archivos HTML, CSS e imágenes.

🧭 ¿Cómo Funciona el Navegador?

El navegador (Chrome, Firefox, Safari, Edge) es un programa que lee archivos y los convierte en algo visual.

1

Escribís una URL o hacés clic en un enlace.

2

El navegador envía un pedido al servidor (request).

3

El servidor responde con los archivos (HTML, CSS, imágenes).

4

El navegador lee el HTML y lo dibuja en pantalla.

¿Qué archivos recibe?

  • HTML — La estructura (qué hay)
  • CSS — El diseño (cómo se ve)
  • Imágenes — Fotos, íconos, logos

Dato curioso 🤓

Podés ver el código de cualquier página web. En Chrome: clic derecho → "Ver código fuente de la página" o Ctrl + U.

🔗 Anatomía de una URL

Cada página web tiene una dirección única llamada URL (Uniform Resource Locator).

Desglosando una URL:

https:// Protocolo seguro
www.bytec.com Dominio (nombre del sitio)
/cursos Ruta (página específica)

Protocolo

https:// significa que la conexión es segura (cifrada). El candadito 🔒 en el navegador lo confirma.

Dominio

Es el "nombre" del sitio. Detrás de cada dominio hay una dirección IP (ej: 142.250.80.46 es Google).

📖 Semántico vs Sintáctico

Dos conceptos fundamentales que aplican tanto al lenguaje humano como al código:

Semántica

El significado. ¿Qué quiere decir?

💭

La semántica se pregunta: "¿Qué significa este contenido?"

vs

Sintaxis

La forma. ¿Cómo se escribe correctamente?

📝

La sintaxis se pregunta: "¿Está bien escrito?"

🗣️ Ejemplo con el español: "Yo gustar pizza" → La semántica se entiende (te gusta la pizza), pero la sintaxis está mal (lo correcto es "A mí me gusta la pizza").

🏷️ Semántica en HTML

En HTML, semántica significa usar la etiqueta correcta según el significado del contenido:

✅ Semántico (correcto)

<h1>Título Principal</h1>
<p>Este es un párrafo.</p>
<nav>Menú de navegación</nav>
<footer>Pie de página</footer>

Cada etiqueta tiene un significado claro. El navegador y los buscadores entienden qué es cada cosa.

❌ No semántico (funciona, pero mal)

<div>Título Principal</div>
<div>Este es un párrafo.</div>
<div>Menú de navegación</div>
<div>Pie de página</div>

Todo es <div> (caja genérica). El navegador no sabe qué es un título y qué es un pie de página.

🔍 ¿Por qué importa? Google usa la semántica para entender tu web. Un <h1> le dice "esto es lo más importante de la página". Un <div> no dice nada.

⚙️ Sintaxis en HTML

La sintaxis son las reglas de escritura. Si no las seguís, el navegador no entiende tu código.

Anatomía de una etiqueta HTML:

<h1> Etiqueta de apertura
══ contenido ══
</h1> Etiqueta de cierre

✅ Sintaxis correcta

<h1>Hola Mundo</h1>
<p>Un párrafo válido.</p>

❌ Sintaxis incorrecta

<h1>Falta el cierre
<p>Esto está mal<p>
h1>Sin el menor que</h1>

📏 Regla de oro: Toda etiqueta que se abre (<tag>) se tiene que cerrar (</tag>). La barra / indica el cierre.

📄 ¿Qué es HTML?

HTML = HyperText Markup Language
(Lenguaje de Marcado de Hipertexto)

¿Lenguaje de programación?

No. HTML no es un lenguaje de programación. Es un lenguaje de marcado: le dice al navegador cómo estructurar el contenido, no le da instrucciones lógicas.

¿Qué es "Hipertexto"?

Texto que contiene enlaces a otros textos. Cuando hacés clic en un link, estás usando hipertexto. Es lo que conecta toda la web.

¿Qué es "Marcado"?

Usar etiquetas (<h1>, <p>) para "marcar" partes del contenido y decirle al navegador: "esto es un título", "esto es un párrafo".

¿Dónde se usa?

En todas las páginas web del mundo. Cada sitio que visitás (Google, Instagram, Wikipedia) está hecho con HTML como base.

🏗️ Estructura Base de HTML

Todo archivo HTML tiene la misma estructura fundamental. Pensalo como el esqueleto de la página:

<!DOCTYPE html>
<html lang="es">
  <head>
    <meta charset="UTF-8">
    <title>Mi Primera Página</title>
  </head>
  <body>
    <!-- Acá va todo lo que se VE en la página -->
  </body>
</html>
<!DOCTYPE html> Le dice al navegador que es HTML5
<html> Raíz de todo el documento
<head> Información invisible (metadatos)
<meta charset> Permite acentos y ñ
<title> Texto de la pestaña del navegador
<body> Todo el contenido visible

🧠 Head vs Body

Las dos secciones principales de un HTML tienen roles muy distintos:

<head>

Lo que el usuario NO ve.

🧠

Metadatos, título de la pestaña, conexión con CSS, idioma, codificación.

vs

<body>

Lo que el usuario SÍ ve.

👀

Títulos, párrafos, imágenes, botones, formularios... todo lo visible.

🎯 Regla simple: Si lo querés ver en la pantalla → va en <body>. Si es configuración o información para el navegador → va en <head>.

🔤 Etiquetas Básicas

Las primeras etiquetas que vas a usar para poner contenido en la página:

Títulos (h1 a h6)

<h1>Título más importante</h1>
<h2>Subtítulo</h2>
<h3>Sub-subtítulo</h3>
<h4>Nivel 4</h4>
<h5>Nivel 5</h5>
<h6>Nivel 6 (más chico)</h6>

Van de <h1> (más importante) a <h6> (menos importante). Usá solo un h1 por página.

Párrafos y saltos

<p>Esto es un párrafo. El navegador
   lo muestra como un bloque de texto
   con espacio arriba y abajo.</p>

<p>Otro párrafo separado.</p>

<br>  <!-- Salto de línea -->
<hr>  <!-- Línea horizontal -->

<p> crea bloques de texto. <br> y <hr> son etiquetas vacías (no necesitan cierre).

💻 ¿Qué es un IDE?

IDE = Integrated Development Environment
(Entorno de Desarrollo Integrado)

Es un editor de texto especializado para escribir código. Es como el Word, pero para programadores.

¿Por qué no usar el Bloc de Notas?

  • No colorea el código (syntax highlighting)
  • No autocompleta etiquetas
  • No marca errores
  • No tiene vista previa

VS Code (el que usamos)

  • Gratuito y creado por Microsoft
  • Colorea el código por tipo
  • Autocompleta etiquetas HTML
  • Extensiones (Live Server, etc.)

⬇️ Descargá VS Code: https://code.visualstudio.com — Es gratis y funciona en Windows, Mac y Linux.

🔧 Configurando tu IDE

Pasos para empezar a trabajar con Visual Studio Code:

1

Creá una carpeta en tu escritorio, ej: mi-primera-web

2

Abrí VS Code → File → Open Folder → elegí tu carpeta.

3

Creá un archivo nuevo: File → New File → nombralo index.html

4

Escribí ! y presioná Tab para generar la estructura HTML automáticamente.

Extensión recomendada: Live Server

Permite ver los cambios en tiempo real en el navegador. Instalala desde el ícono de extensiones en VS Code buscando "Live Server".

¿Por qué "index.html"?

Por convención, index.html es la página principal de un sitio. Cuando entrás a un dominio, el servidor busca este archivo automáticamente.

⚡ El Truco del "!" (Emmet)

VS Code tiene un atajo increíble. En un archivo .html vacío, escribí ! y presioná Tab:

<!DOCTYPE html>
<html lang="en">
<head>
    <meta charset="UTF-8">
    <meta name="viewport" content="width=device-width, initial-scale=1.0">
    <title>Document</title>
</head>
<body>

</body>
</html>

¡Se genera toda la estructura base automáticamente! Solo tenés que:

✏️ Cambiar lang="en" por lang="es" (estamos en español)
✏️ Cambiar el <title> por el nombre de tu página
✏️ Empezar a escribir tu contenido dentro del <body>

💻 Práctica: ¡Tu Primera Página Web!

Vamos a codear juntos. Seguí estos pasos:

📁 Creá una carpeta mi-primera-web
📄 Dentro, creá un archivo index.html
⚡ Escribí ! + Tab para la estructura base
✏️ Cambiá el idioma a es y el título
🔤 Dentro del body, agregá un <h1>, un <h2> y dos <p>
<!DOCTYPE html>
<html lang="es">
<head>
    <meta charset="UTF-8">
    <meta name="viewport" content="width=device-width, initial-scale=1.0">
    <title>Mi Primera Página</title>
</head>
<body>
    <h1>¡Hola Mundo!</h1>
    <h2>Mi primera página web</h2>
    <p>Esto es un párrafo. Estoy aprendiendo HTML en BYTEC.</p>
    <p>Ya sé que HTML es un lenguaje de marcado, no de programación.</p>
</body>
</html>

👉 Guardá el archivo (Ctrl + S) y abrilo en el navegador haciendo doble clic, o usá Live Server.

🧩 Más Etiquetas Útiles

Además de títulos y párrafos, HTML tiene etiquetas para listas, texto en negrita y más:

Listas

<!-- Lista con viñetas -->
<ul>
  <li>HTML</li>
  <li>CSS</li>
  <li>JavaScript</li>
</ul>

<!-- Lista numerada -->
<ol>
  <li>Primero</li>
  <li>Segundo</li>
  <li>Tercero</li>
</ol>

Formato de texto

<!-- Negrita -->
<strong>Texto importante</strong>

<!-- Cursiva -->
<em>Texto enfatizado</em>

<!-- Enlace -->
<a href="https://google.com">
  Ir a Google
</a>

<!-- Imagen -->
<img src="foto.jpg" alt="Descripción">

💡 <ul> = Unordered List (sin números). <ol> = Ordered List (con números). <li> = List Item (cada elemento).
