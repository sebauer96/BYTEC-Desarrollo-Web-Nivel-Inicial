📦 Todo es una Caja

En CSS, cada elemento HTML es un rectángulo (una caja). No importa si es un título, un párrafo, un botón o una imagen.

Lo que vos ves

📝

Texto, imágenes, botones con formas variadas.

vs

Lo que el navegador ve

⬜⬜⬜

Rectángulos apilados uno debajo del otro.

🔍 Probalo ahora: Abrí cualquier web, hacé clic derecho → "Inspeccionar" y pasá el mouse sobre los elementos. Vas a ver los rectángulos resaltados en colores.

🏗️ El Box Model (Modelo de Caja)

Cada caja tiene 4 capas, de adentro hacia afuera:

MARGIN
BORDER
PADDING
CONTENT
Content
El contenido real (texto, imagen)
Padding
Espacio INTERNO (entre contenido y borde)
Border
El borde visible de la caja
Margin
Espacio EXTERNO (entre esta caja y las demás)

📝 1. Content (Contenido)

Es la capa más interna: el texto, la imagen o cualquier contenido del elemento. Se controla con width y height.

.caja {
  width: 300px;    /* Ancho del contenido */
  height: 200px;   /* Alto del contenido */
}

Unidades de medida

  • px — Píxeles (tamaño fijo)
  • % — Porcentaje del padre
  • vw / vh — % del viewport
  • auto — El navegador decide

¿Y si no pongo width/height?

Por defecto, un elemento bloque ocupa el 100% del ancho disponible y su alto se adapta al contenido. No siempre es necesario definirlos.

🧸 2. Padding (Espacio Interno)

El padding agrega espacio entre el contenido y el borde. Es como la "espuma" dentro de una caja de envío.

/* Mismo padding en los 4 lados */
.caja {
  padding: 20px;
}

/* Diferente por lado: arriba, derecha, abajo, izquierda (sentido reloj 🕐) */
.caja {
  padding: 10px 20px 10px 20px;
}

/* Atajo: vertical | horizontal */
.caja {
  padding: 10px 20px;    /* 10 arriba/abajo, 20 izq/der */
}

/* Por lado individual */
.caja {
  padding-top: 10px;
  padding-right: 20px;
  padding-bottom: 10px;
  padding-left: 20px;
}

🕐 Truco mnemotécnico: El orden de los 4 valores es como un reloj: Top → Right → Bottom → Left (arriba → derecha → abajo → izquierda).

🔲 3. Border (Borde)

El border es la línea visible que rodea al padding. Tiene 3 propiedades: grosor, estilo y color.

/* Atajo: grosor estilo color */
.caja {
  border: 2px solid #f75001;
}

/* Por propiedad */
.caja {
  border-width: 2px;
  border-style: solid;
  border-color: #f75001;
}

/* Solo un lado */
.caja {
  border-bottom: 3px dashed #04589d;
}

/* Redondear esquinas */
.caja {
  border-radius: 10px;     /* Todas las esquinas */
  border-radius: 50%;      /* Círculo perfecto */
}
Estilo Descripción Ejemplo
solid Línea continua (el más usado) border: 2px solid red
dashed Línea de guiones border: 2px dashed blue
dotted Línea de puntos border: 2px dotted green
none Sin borde border: none

↔️ 4. Margin (Espacio Externo)

El margin agrega espacio fuera del borde, separando la caja de los demás elementos. Funciona igual que el padding:

/* Mismo margin en los 4 lados */
.caja {
  margin: 20px;
}

/* Vertical | Horizontal */
.caja {
  margin: 20px 40px;
}

/* Centrar horizontalmente un bloque con ancho fijo */
.caja {
  width: 600px;
  margin: 0 auto;    /* 0 arriba/abajo, AUTO a los costados = centrado */
}

Padding vs Margin

  • Padding: espacio DENTRO (tiene fondo)
  • Margin: espacio FUERA (transparente)
  • Padding empuja el contenido hacia adentro
  • Margin empuja otros elementos hacia afuera

El truco de margin: 0 auto

Es la forma clásica de centrar un bloque horizontalmente. Necesita un width definido para funcionar. El auto reparte el espacio sobrante de forma igual a cada lado.

🧮 ¿Cuánto mide realmente una caja?

Por defecto, el navegador usa content-box: el width solo aplica al contenido. Padding y border se suman.

.caja {
  width: 300px;
  padding: 20px;        /* 20 × 2 = 40px extra */
  border: 5px solid;    /*  5 × 2 = 10px extra */
}

/* Ancho REAL en pantalla = 300 + 40 + 10 = 350px 😱 */

Fórmula del ancho real (content-box):

width 300px
+
padding × 2 40px
+
border × 2 10px
=
350px Ancho real

😤 Esto es confuso y causa problemas de layout. Por eso existe border-box...

✨ box-sizing: border-box

Con border-box, el width incluye padding y border. El navegador ajusta el contenido para que todo entre en el tamaño que definiste.

content-box (por defecto)

width: 300px

+ padding + border = 350px reales

😤
vs

border-box (recomendado)

width: 300px

padding y border incluidos = 300px reales

😎
/* ⭐ Buena práctica: aplicar a TODOS los elementos */
* {
  box-sizing: border-box;
}

/* Con esto, width: 300px SIEMPRE será 300px en pantalla,
   sin importar el padding o border que le pongas */

⭐ Regla universal: Siempre agregá * { box-sizing: border-box; } al inicio de tu CSS. Es lo primero que hacen todos los desarrolladores profesionales.

📏 Width, Height y sus variantes

Además de width y height, tenemos propiedades para limitar tamaños:

Propiedad ¿Qué hace? Ejemplo
width Ancho fijo del elemento width: 400px
height Alto fijo del elemento height: 200px
max-width Ancho máximo (se achica si la pantalla es menor) max-width: 600px
min-height Alto mínimo (crece si el contenido lo necesita) min-height: 100px
width: 100% Ocupa todo el ancho disponible del padre width: 100%

💡 Tip profesional: Usá max-width en vez de width fijo cuando sea posible. Así tu diseño se adapta mejor a pantallas chicas.

🧱 Display: Block vs Inline

Cada elemento HTML tiene un comportamiento por defecto que define cómo ocupa espacio:

display: block

Ocupa todo el ancho disponible. Cada elemento empieza en una nueva línea.

⬜⬜⬜⬜⬜⬜
⬜⬜⬜⬜⬜⬜
⬜⬜⬜⬜⬜⬜

h1, p, div, form, ul, li

vs

display: inline

Ocupa solo lo necesario. Se pone al lado del elemento anterior.

⬜ ⬜ ⬜ ⬜ ⬜

a, strong, em, span, code

/* Cambiar el comportamiento */
span {
  display: block;     /* Ahora ocupa toda la línea */
}

li {
  display: inline;    /* Ahora se ponen uno al lado del otro */
}

🧩 display: inline-block

Es un híbrido: se comporta como inline (se pone al lado), pero permite definir width, height, padding y margin como un block.

.etiqueta {
  display: inline-block;
  background-color: #04589d;
  color: white;
  padding: 8px 16px;
  margin: 4px;
  border-radius: 20px;
}
Propiedad ¿Se pone al lado? ¿Acepta width/padding?
block ❌ No, va en nueva línea ✅ Sí
inline ✅ Sí, al lado ❌ No (width/height ignorados)
inline-block ✅ Sí, al lado ✅ Sí
none — Elemento invisible — No se muestra

🌊 Overflow: Contenido que se desborda

¿Qué pasa cuando el contenido es más grande que la caja? La propiedad overflow lo controla:

.caja {
  width: 300px;
  height: 150px;
  overflow: hidden;     /* Corta el contenido que no entra */
}

.caja-scroll {
  width: 300px;
  height: 150px;
  overflow: auto;       /* Agrega scroll si es necesario */
}
visible
(por defecto) El contenido se sale de la caja
hidden
Corta/oculta lo que no entra
auto
Agrega scrollbar solo si hace falta

💡 Tip: overflow: auto es el más flexible. Solo muestra scrollbar cuando realmente se necesita.

⚡ Shorthands (Atajos CSS)

CSS permite escribir varias propiedades en una sola línea. Acá los atajos más útiles:

Margin / Padding

/* 1 valor: los 4 lados iguales */
margin: 20px;

/* 2 valores: vertical | horizontal */
margin: 10px 20px;

/* 3 valores: top | horizontal | bottom */
margin: 10px 20px 30px;

/* 4 valores: top right bottom left */
margin: 10px 20px 30px 40px;

Border

/* Atajo completo: ancho estilo color */
border: 2px solid #f75001;

/* Es lo mismo que escribir: */
border-width: 2px;
border-style: solid;
border-color: #f75001;

/* Solo un lado */
border-left: 4px solid blue;
border-bottom: 1px dashed gray;

🧠 Mnemotécnico de 4 valores: Pensá en un reloj → empieza a las 12 (top), sigue a las 3 (right), luego 6 (bottom) y termina en 9 (left).
