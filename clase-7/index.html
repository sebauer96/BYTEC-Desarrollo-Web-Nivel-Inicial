🏗️ ¿Qué es Maquetar?

Maquetar = traducir un diseño visual a código HTML + CSS funcional. Es lo que hace un desarrollador frontend cuando recibe un mockup:

🎨

Diseño

Figma, Photoshop o un dibujo en papel.

→
🛠️

Maquetación

HTML estructura + CSS estilos = web funcional.

→
🌐

Sitio web

Lo ven los usuarios en el navegador.

🎯 Tu trabajo como dev frontend: que el sitio en el navegador se vea idéntico al diseño, sea responsive y funcione en todos los navegadores.

🏛️ Estructura Semántica de un Sitio

Antes de escribir código, pensá en las secciones de tu página. HTML5 tiene etiquetas semánticas para cada zona:

<header> — Logo + Navegación
<section class="hero"> — Bienvenida
<section class="services"> — Servicios
<section class="gallery"> — Trabajos
<section class="contact"> — Contacto
<footer> — Copyright + Links

📚 ¿Por qué semántico? Google entiende mejor tu sitio (SEO), los lectores de pantalla ayudan a personas con discapacidad visual, y tu código queda autoexplicativo.

🏷️ Etiquetas Semánticas Clave

Estas son las etiquetas que vas a usar para estructurar cualquier sitio web:

Etiqueta ¿Qué representa? Ejemplo
<header> Encabezado del sitio (logo + nav) Top de la página
<nav> Bloque de navegación Menu principal
<main> Contenido principal único Una vez por página
<section> Sección temática del contenido Hero, servicios, etc.
<article> Pieza de contenido autónoma Post de blog, card
<aside> Contenido lateral relacionado Sidebar, widget
<footer> Pie de página Copyright, links finales

📝 Paso 1: HTML Semántico Base

Empezá siempre por el HTML. Sin estilos, solo la estructura:

<!DOCTYPE html>
<html lang="es">
<head>
  <meta charset="UTF-8">
  <meta name="viewport" content="width=device-width, initial-scale=1.0">
  <title>Pixel Café — Coffee for Gamers</title>
  <link rel="stylesheet" href="style.css">
</head>
<body>

  <header class="site-header">
    <a href="#" class="logo">🎮 PIXEL CAFÉ</a>
    <nav>...</nav>
  </header>

  <main>
    <section class="hero">...</section>
    <section class="menu">...</section>
    <section class="gallery">...</section>
    <section class="about">...</section>
  </main>

  <footer class="site-footer">...</footer>

</body>
</html>

⭐ Regla de oro: escribí PRIMERO toda la estructura HTML, después agregás los estilos CSS. No mezcles los dos pasos.

🧭 Paso 2: Header + Navegación

El header es lo primero que ve el usuario. Tiene que ser simple y dejar claro qué es el sitio:

<!-- HTML -->
<header class="site-header">
  <a href="#" class="logo">🎮 PIXEL CAFÉ</a>
  <nav>
    <ul class="nav-links">
      <li><a href="#menu">Menú</a></li>
      <li><a href="#galeria">Galería</a></li>
      <li><a href="#sobre">Sobre</a></li>
      <li><a href="#contacto">Contacto</a></li>
    </ul>
  </nav>
</header>
/* CSS — Mobile first: stacked, después flex en desktop */
.site-header {
  display: flex;
  flex-direction: column;
  align-items: center;
  padding: 16px;
  background: var(--dark-surface);
  gap: 12px;
}

@media (min-width: 768px) {
  .site-header {
    flex-direction: row;
    justify-content: space-between;
    padding: 0 32px;
    height: 70px;
  }
}

El hero es la primera sección visible. Tiene que enganchar al usuario en 3 segundos:

📂 Organizar tu CSS

Cuando un CSS crece, se vuelve un caos. Organizalo siempre con esta estructura:

/* ============================================
   1. RESET & VARIABLES
   ============================================ */
* { margin: 0; padding: 0; box-sizing: border-box; }
:root { --color-primario: #f75001; ... }

/* ============================================
   2. BASE (body, fuentes, links)
   ============================================ */
body { font-family: 'Inter', sans-serif; ... }

/* ============================================
   3. LAYOUT (header, footer, secciones)
   ============================================ */
.site-header { ... }
.hero { ... }
.menu { ... }

/* ============================================
   4. COMPONENTES (botones, cards, formularios)
   ============================================ */
.btn { ... }
.menu-card { ... }

/* ============================================
   5. UTILIDADES (clases helper)
   ============================================ */
.text-center { text-align: center; }

/* ============================================
   6. RESPONSIVE (media queries al final)
   ============================================ */
@media (min-width: 768px) { ... }

🧱 BEM: Nomenclatura para Clases

BEM (Block-Element-Modifier) es una convención muy usada para nombrar clases CSS. Te ayuda a evitar conflictos:

/* Block: el componente principal */
.card { ... }

/* Element: una parte del block (separado por __) */
.card__title { ... }
.card__image { ... }
.card__button { ... }

/* Modifier: variante del block o element (con --) */
.card--featured { ... }       /* Una card destacada */
.card__button--disabled { ... } /* Un botón deshabilitado */

Ventajas de BEM

  • Sin conflictos entre clases
  • Fácil saber a qué pertenece cada clase
  • Componentes reusables y predecibles
  • Sin selectores anidados complejos

Si BEM te parece mucho

Para proyectos chicos podés usar nombres simples como .menu-card o .btn-primary. Lo importante es ser consistente en todo el proyecto.
