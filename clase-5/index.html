🔄 Repaso Rápido — Clase 4

En la clase anterior aprendimos Flexbox para crear layouts profesionales:

display: flex Activa Flexbox
justify-content Eje principal
align-items Eje cruzado
flex-wrap Saltar de línea
gap Espacio entre hijos
flex: 1 Crecer flexible

💡 Flexbox nos permite alinear y distribuir elementos. Hoy vamos a hacer que esos layouts se adapten automáticamente al tamaño de cada pantalla.

📱 ¿Por Qué Responsive?

Hoy más del 60% del tráfico web viene de celulares. Tu web tiene que verse bien en todas las pantallas:

Celular
~375px
Tablet
~768px
Escritorio
~1200px+

🎯 Responsive Design = un solo código HTML/CSS que se adapta a cualquier pantalla. No necesitás hacer una web diferente para cada dispositivo.

🔍 El Viewport Meta Tag

Para que el diseño responsive funcione en celulares, siempre necesitás esta línea en el <head>:

<meta name="viewport" content="width=device-width, initial-scale=1.0">

Sin viewport meta

El celular muestra la web como si fuera una pantalla de escritorio. Todo se ve diminuto.

🔬
vs

Con viewport meta

El celular usa su ancho real. El CSS puede adaptarse correctamente.

📱✅

⭐ Regla de oro: Si no tenés este meta tag, tus media queries no van a funcionar en celulares. ¡Siempre incluilo!

📏 Unidades Relativas vs Absolutas

Para diseños responsive, preferí unidades relativas que se adaptan al contexto:

Unidad ¿Relativa a qué? Ejemplo
px Nada (absoluta, fija) width: 400px
% Al tamaño del elemento padre width: 80%
em Al font-size del elemento padre padding: 1.5em
rem Al font-size del <html> (root) font-size: 1.2rem
vw 1% del ancho de la ventana width: 50vw
vh 1% del alto de la ventana height: 100vh

💡 Tip: Usá rem para font-sizes, % para anchos de contenedores y px solo para cosas que deben ser fijas (bordes, sombras).

📐 El Combo: max-width + %

Antes de las media queries, podés hacer mucho solo con unidades flexibles:

/* ❌ Ancho fijo — se rompe en pantallas chicas */
.contenedor {
  width: 900px;
}

/* ✅ Ancho flexible — se adapta */
.contenedor {
  width: 90%;            /* Ocupa el 90% del padre */
  max-width: 900px;      /* Pero nunca más de 900px */
  margin: 0 auto;        /* Centrado horizontal */
}

/* ✅ Imágenes responsive */
img {
  max-width: 100%;       /* Nunca más ancha que su padre */
  height: auto;          /* Mantiene la proporción */
}

width: 90%

Se adapta al padre. En un celular de 375px, el contenedor será de 337px. En desktop, 90% del viewport.

max-width: 900px

Pone un límite. En pantallas grandes, no crece más de 900px. El contenido no se estira demasiado.

🎛️ Media Queries: ¿Qué Son?

Las media queries son bloques de CSS que solo se aplican cuando se cumple una condición (generalmente el ancho de la pantalla):

/* Estilos base — se aplican SIEMPRE */
.titulo {
  font-size: 18px;
  color: white;
}

/* Solo se aplica cuando la pantalla es de 768px o más */
@media (min-width: 768px) {
  .titulo {
    font-size: 28px;    /* Más grande en pantallas medianas */
  }
}

/* Solo se aplica cuando la pantalla es de 1200px o más */
@media (min-width: 1200px) {
  .titulo {
    font-size: 36px;    /* Aún más grande en desktop */
  }
}

🧠 Pensalo así: @media es como un "si la pantalla mide X, aplicá estos estilos". Es como un if de CSS.

📝 Sintaxis de @media

La estructura de una media query:

KEYWORD @media
CONDICIÓN (min-width: 768px)
{
CSS RULES /* estilos aquí */
}
Condición ¿Cuándo se activa? Uso
min-width Pantalla ≥ al valor Mobile first ⭐
max-width Pantalla ≤ al valor Desktop first
combinadas Rango entre dos valores (min: 768) and (max: 1199)

⭐ La industria recomienda usar min-width (mobile first). Empezás diseñando para celular y después agregás estilos para pantallas más grandes.

📊 Breakpoints Comunes

Los breakpoints son los anchos donde tu diseño cambia. Estos son los más usados:

Mobile 0 – 767px
Tablet 768 – 1023px
Desktop 1024px+ 
/* Mobile first: estilos base para celular */
.container { padding: 16px; }

/* Tablet: 768px en adelante */
@media (min-width: 768px) {
  .container { padding: 24px; max-width: 720px; margin: 0 auto; }
}

/* Desktop: 1024px en adelante */
@media (min-width: 1024px) {
  .container { padding: 32px; max-width: 960px; }
}

/* Desktop grande: 1200px en adelante */
@media (min-width: 1200px) {
  .container { max-width: 1140px; }
}

💡 No hay breakpoints "correctos" universales. Estos son los más comunes, pero lo ideal es adaptar según tu diseño.

📱 Mobile First: La Estrategia

Mobile first = empezás diseñando para la pantalla más chica y después agregás estilos para pantallas más grandes:

Desktop First ❌

/* Base: desktop */
.nav { display: flex; }

/* Achicás para mobile */
@media (max-width: 768px) {
  .nav { flex-direction: column; }
}

Más código, más complejo, cargás cosas innecesarias en mobile.

vs

Mobile First ✅

/* Base: mobile */
.nav { display: flex; flex-direction: column; }

/* Expandís para desktop */
@media (min-width: 768px) {
  .nav { flex-direction: row; }
}

Más simple, más eficiente, mejor rendimiento en mobile.

🎯 ¿Por qué mobile first? Porque es más fácil agregar complejidad que sacarla. Empezás simple y después ampliás.

🛠️ Ejemplo: Cards Responsive

Veamos cómo hacer que unas cards pasen de 1 columna en mobile a 3 columnas en desktop:

/* BASE (mobile): cards apiladas verticalmente */
.cards {
  display: flex;
  flex-direction: column;
  gap: 16px;
}

.card {
  background: #12123a;
  padding: 24px;
  border-radius: 12px;
  border: 1px solid #1a1a5e;
}

/* TABLET (768px+): 2 columnas */
@media (min-width: 768px) {
  .cards {
    flex-direction: row;
    flex-wrap: wrap;
  }
  .card {
    flex: 1 1 calc(50% - 8px);   /* 2 cards por fila */
  }
}

/* DESKTOP (1024px+): 3 columnas */
@media (min-width: 1024px) {
  .card {
    flex: 1 1 calc(33.333% - 11px); /* 3 cards por fila */
  }
}

🔑 El truco: Combinás flex-wrap: wrap con flex-basis calculado. En mobile va en columna, en tablet de a 2, y en desktop de a 3.

💻 Práctica 1: Primeras Media Queries

Vamos a crear una página simple que cambie de colores y layout según el tamaño de pantalla:

<div class="banner">
  <h1>Responsive Design</h1>
  <p>Achicá la ventana para ver los cambios</p>
</div>
<div class="contenido">
  <div class="box">Caja 1</div>
  <div class="box">Caja 2</div>
  <div class="box">Caja 3</div>
</div>
/* Mobile first: base */
.banner { background: #f75001; padding: 20px; text-align: center; }
.contenido { display: flex; flex-direction: column; gap: 12px; padding: 16px; }
.box { background: #12123a; padding: 24px; border-radius: 8px; text-align: center; }

/* Tablet */
@media (min-width: 768px) {
  .contenido { flex-direction: row; }
  .box { flex: 1; }
  .banner { padding: 40px; }
}
📱 Achicá la ventana del navegador y observá el cambio
🎨 Agregá un @media que cambie el fondo del banner en desktop
📏 Usá el inspector (F12) → click en el ícono de celular para simular dispositivos

🔄 Patrones Responsive Comunes

Hay patrones que se repiten en casi todas las webs responsive:

📚

Stack → Row

En mobile se apilan vertical, en desktop van en fila.

.items { flex-direction: column; }
@media (min-width: 768px) {
  .items { flex-direction: row; }
}
🙈

Mostrar / Ocultar

Elementos que se ocultan en mobile y aparecen en desktop.

.sidebar { display: none; }
@media (min-width: 1024px) {
  .sidebar { display: block; }
}
🔤

Tamaño de Texto

Texto más chico en mobile, más grande en desktop.

h1 { font-size: 24px; }
@media (min-width: 768px) {
  h1 { font-size: 36px; }
}
@media (min-width: 1024px) {
  h1 { font-size: 48px; }
}
📐

Cambiar Padding

Menos espacio en mobile, más generoso en desktop.

section { padding: 16px; }
@media (min-width: 768px) {
  section { padding: 40px; }
}
@media (min-width: 1024px) {
  section { padding: 60px 80px; }
}

🖼️ Imágenes Responsive

Las imágenes pueden romper tu layout si son más anchas que la pantalla. La solución es simple:

/* ⭐ Regla universal para imágenes responsive */
img {
  max-width: 100%;     /* Nunca más ancha que su contenedor */
  height: auto;        /* Mantiene la proporción */
  display: block;      /* Evita espacio extra debajo */
}

/* Imagen que siempre ocupa todo el ancho */
.hero-img {
  width: 100%;
  height: 300px;
  object-fit: cover;   /* Recorta para llenar sin distorsionar */
}

/* Imagen con tamaño diferente según pantalla */
.logo {
  width: 120px;
}

@media (min-width: 768px) {
  .logo {
    width: 180px;
  }
}

max-width: 100%

La imagen se achica si el contenedor es más chico, pero nunca se estira más allá de su tamaño original.

object-fit: cover

Para imágenes con tamaño fijo: recorta la imagen para que llene el espacio sin distorsionarse.

🍔 Navbar Responsive

Un patrón super común: navbar horizontal en desktop, menú apilado en mobile:

/* BASE (mobile): links apilados */
.navbar {
  display: flex;
  flex-direction: column;
  padding: 16px;
  background: #0d0d2b;
}

.navbar .logo {
  margin-bottom: 12px;
}

.nav-links {
  display: flex;
  flex-direction: column;
  gap: 8px;
  list-style: none;
}

/* DESKTOP (768px+): todo en una fila */
@media (min-width: 768px) {
  .navbar {
    flex-direction: row;
    align-items: center;
    padding: 0 24px;
    height: 60px;
  }

  .navbar .logo {
    margin-bottom: 0;
  }

  .nav-links {
    flex-direction: row;
    flex: 1;
    justify-content: flex-end;
    gap: 24px;
  }
}

💡 En una web real, la versión mobile tendría un botón hamburguesa (☰) que muestra/oculta el menú. Eso requiere JavaScript, que veremos en otro momento.
